# 데이터 유지 (배포·푸시 시)

## 요약
- **배포(커밋/푸시) 후 재배포해도 DB에 저장된 내용은 유지됩니다.** (Railway 등에서 `DATABASE_URL`로 Postgres 사용 시)
- 정책표 수정, 대리점, 고객 신청·견적, 이메일 로그 등은 **모두 DB에만 저장**되며, Git 저장소나 엑셀 파일로 덮어쓰지 않습니다.

## 정책표 (PolicyRow)
- **수정한 정책표 데이터**는 DB에 저장됩니다. 커밋/푸시는 **코드만** 올라가며, DB는 서버의 Postgres에 그대로 유지됩니다.
- **자동 불러오기**  
  - **로컬(SQLite)**: 앱 최초 실행 시 정책 행이 0건이면 `app/data/moson_policy.xlsx`가 있을 때만 한 번 자동 불러오기.  
  - **배포(Postgres, `DATABASE_URL` 설정 시)**: 앱 시작 시 **정책 자동 불러오기를 하지 않습니다.**  
    → 재배포 후에도 정책표에서 수정해 둔 내용이 그대로 유지됩니다.
- **엑셀에서 다시 불러오고 싶을 때**  
  - 어드민 **정책표** 페이지에서 **「업로드한 엑셀로 불러오기」** 또는 **「서버 파일 불러오기」**를 **직접** 눌렀을 때만, 기존 정책 데이터가 엑셀 내용으로 **전체 교체**됩니다.  
  - 정책표 페이지를 그냥 **열기만** 해서 자동으로 불러오기는 되지 않습니다.

## 그 외 테이블 (대리점, 견적/신청, 이메일 로그 등)
- **Reseller, ConsultRequest, VisitLog, EmailLog, AdminUser, ConsultStatus, ResellerApplication**  
  - 앱 시작 시 **삭제·비우기·덮어쓰기 없음.**  
  - 조회·추가·수정·개별 삭제만 있으며, 배포와 무관하게 DB에 유지됩니다.
- 대리점 삭제는 “삭제 대기” 후 24시간 지나면 삭제되는 **비즈니스 로직**만 있으며, 배포 시점에 테이블을 비우는 동작은 없습니다.

## 배포 시 유의사항
- **반드시 `DATABASE_URL`(Postgres)을 설정**해 두세요.  
  설정되지 않으면 컨테이너 안 SQLite를 쓰게 되고, 재배포 시 컨테이너가 새로 만들어지면 **DB 파일이 초기화**될 수 있습니다.
- Railway 등에서 Postgres 추가 후 `DATABASE_URL`만 연결해 두면, 위 모든 데이터는 재배포 후에도 **최신 내용이 유지**됩니다.
